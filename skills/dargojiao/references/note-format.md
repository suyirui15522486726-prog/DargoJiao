# 笔记结构与排版

## 固定结构

每条笔记按以下顺序生成：

1. `连续序号｜标题`
2. 总结时间，使用配置时区
3. 面试问题 Topic
4. 错误答题
5. 高分答题
6. 结合项目谈一谈，仅在存在直接代码证据时出现
7. 视频来源与去重信息

## 内容口径

- 能取得视频逐字证据时，忠实提炼视频中的低分与高分回答。
- 只能核验题目和技术事实时，把第四节命名为“错误答题（反例归纳）”，并说明不是视频原话。
- 无法确认同一作品或核心观点时，不创建笔记。
- 高分回答采用“先结论、再原理、再边界、最后落地”的面试表达，不写普通视频摘要。

## 飞书格式

- 每条笔记使用二级标题，内部小节使用三级标题。
- 总结时间和 Topic 放在引用块或醒目的摘要区域。
- 对比项使用列表；仅在信息需要横向比较时使用表格。
- 项目代码必须放在带语言标记的代码块中，并在代码块前标注仓库相对路径和源码符号。
- 原视频使用可点击链接。
- 去重标识放在末尾的灰色处理信息中，不影响正文阅读。

## 项目证据模板

```text
项目代码位置：<仓库相对路径>
关键符号：<类、方法或配置键>
项目中的作用：<知识点如何实际落地>
```

代码块只截取支撑结论所需的最小片段，不粘贴整份源文件。
