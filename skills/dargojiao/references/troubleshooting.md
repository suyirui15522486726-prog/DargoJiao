# 故障分类与处理

| 阶段 | 常见现象 | Codex 应采取的动作 | 用户动作 |
|---|---|---|---|
| 飞书授权 | 用户身份无效或授权过期 | 停止写入并保留待重试 | 重新完成 `lark-cli` 用户登录 |
| 群消息 | 找不到群或无历史消息权限 | 核对群名与当前账号可见性 | 确认账号已入群并允许读取历史消息 |
| Wiki | 能打开链接但不能创建或编辑 | 区分接口权限和知识库成员权限 | 把当前用户加入知识空间并授予编辑权限 |
| 链接解析 | 只收到标题、截图、token 或不完整 URL | 不猜测 Wiki/Docx 目标 | 从飞书目标页面重新复制完整链接 |
| 网络或代理 | 短链、飞书或 GitHub 超时 | 不推进状态，记录失败域名和阶段 | 恢复网络或代理后重试 |
| 视频证据 | 只能确认标题，无法确认观点 | 标为待重试，不猜测 | 无需上传视频，等待后续可访问来源 |
| 本地项目 | 路径、ref 或源码不存在 | 省略项目段或报告配置错误 | 修正项目路径和 Git ref |
| 文档写入 | 追加成功状态不明确 | 先回读唯一键，再决定是否重写 | 通常无需手工删除内容 |
| 运行结果 | 未看到本次报告 | 检查 Codex 已安排任务状态和最近运行 | 确认电脑唤醒且 Codex 正在运行 |

排查网络时分别检查 Codex/OpenAI、GitHub/npm、飞书 OAuth/API 和短视频跳转，不要因为其中一条可用就假设其他链路也可用。Windows 的系统代理、浏览器代理、WinHTTP 和 PowerShell 环境变量可能不同。

异常结果使用以下结构：

```text
⚠️ 短视频面试笔记执行异常
时间：<本地时间>
阶段：<授权、网络、消息、证据、项目、写入或回读>
原因：<不含凭据的简短原因>
操作：<用户下一步>
重试：未处理链接会在下一次计划运行中继续检查
```
