# DargoJiao 视频证据门槛修复设计

## 背景与根因

当前规则把“公开页面没有正文或字幕”直接归类为“证据不足”，即使短链已经解析出稳定作品 ID、同条飞书分享也包含作者和完整题目，仍会停止生成笔记。这把作品身份确认、题目识别和视频原话还原错误地合并成一个门槛，导致正常抖音分享长期待重试。

## 目标

- 正常分享在作品 ID 和题目明确时必须进入笔记流程，公开页缺少正文或字幕本身不能成为待重试理由。
- 不把通用技术知识伪装成视频逐字内容。
- 保留写入失败、权限失败、身份冲突和题目不明确时的补处理机制。
- 继续保持纯链接、无下载器、无 FFmpeg、无本地语音模型的轻量架构。

## 分层判定

### 1. 作品身份

解析短链并取得平台作品 ID，以平台和作品 ID 作为视频主身份。同条飞书消息中的作者、标题和话题是身份元数据与冲突检查依据。若作品 ID 缺失，或解析结果与分享元数据明显冲突，保留待重试。

### 2. 题目识别

完整分享标题能够表达清晰面试问题或技术 Topic 时，即可进入笔记生成。标题过短、只有封面、只有作者或无法识别技术题目时，才因题目不足待重试。

### 3. 回答依据

- 能取得字幕、页面正文或同作品可信镜像时，可以归纳视频观点，但仍避免未经核实的逐字引用。
- 只能取得作品 ID 和清晰题目时，使用可靠技术资料与可选本地项目代码生成高分回答；错误答题必须标为“反例归纳”，并说明不是视频原话。
- 技术事实本身无法核验时保留待重试，不用猜测填充。

## 状态与去重

只有飞书文档写入成功且回读验证通过后，才写入 `svnote-msg:<message_id>`、规范化 URL 和平台作品 ID 对应的成功标识。待重试项不占序号、不写成功标识，下一次扫描最近 7 天消息时继续补处理。

## 实际自动化同步

更新现有 `automation-2`，保持工作日 09:30、OneCoupon `main`、最近 7 天和 Codex「已安排」报告方式不变。移除旧的“无正文即拒绝处理”语义，不恢复系统通知或飞书群主动回执。

## 验证

- 回归测试必须证明公开规则包含“作品 ID + 清晰题目仍归档”和“缺正文/字幕不单独待重试”。
- 回归测试必须证明身份缺失、题目不足、元数据冲突和技术事实不可核验仍会待重试。
- 仓库验证、Python 单元测试、Shell 语法、模板副本一致性、Ubuntu CI 和 Windows CI 全部通过。
- 修复按语义化版本发布为 `v0.2.1`，保留 `v0.1.0` 与 `v0.2.0`。
