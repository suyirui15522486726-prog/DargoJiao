# 安全说明

## 凭据

- 飞书凭据保存在 `lark-cli` 的本地凭据存储中。
- DargoJiao 仓库、Skill、计划任务提示词和文档都不保存访问令牌。
- 不在 Issue、截图或运行日志中公开群 ID、用户 ID、Wiki token、授权 URL 参数或应用密钥。

## 本地项目

- 只读取用户明确指定的项目和 Git ref。
- 不切换分支、不修改文件、不提交项目代码。
- 写入飞书的项目证据只包含必要的相对路径、符号和最小代码片段。

## 内容获取

- 默认直接访问分享链接和公开页面。
- 短链解析出的平台作品 ID 用于确认作品身份，作者与精确标题用于补充身份和发现冲突。
- 不读取浏览器凭据、不下载视频、不上传用户媒体、不调用外部内容解析服务。
- 作品身份与题目明确时，公开页缺少正文或字幕本身不能成为待重试理由；允许基于分享题目与可核验技术事实整理答案。
- 没有逐字证据时必须标注“反例归纳”和内容来源，不用通用知识冒充视频原话；身份、题目或技术事实无法核验时保留待重试。

## 自动化

- 使用满足飞书访问和项目只读分析的最小权限。
- 异常报告不得包含 token、Cookie、个人本地路径或代理地址。
- Codex「已安排」中的异常报告只显示失败阶段和用户动作，不显示资源标识。

## 发布检查

`python3 tests/validate_repo.py` 会扫描公开文本中的个人绝对路径、飞书资源 ID、代理端口、常见令牌和私钥头。发布前还应人工检查 Git 历史和跟踪文件列表。
