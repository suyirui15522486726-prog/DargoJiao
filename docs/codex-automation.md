# Codex 自动化

## 为什么必须使用 Codex

DaGoJiao 把内容访问、证据判断、面试笔记生成、源码关联和 Wiki 路由交给 Codex。仓库只提供 Skill、模板和诊断工具，不运行独立后台服务。

Codex 官方文档说明，计划任务可以使用本地项目与 Skill，并在「已安排」中保留每次运行结果。参见 [Scheduled tasks](https://learn.chatgpt.com/docs/automations)。

## 创建规则

- 一个工作流只保留一个计划任务。
- 使用独立运行任务，让每次结果出现在「已安排」。
- 任务关联需要只读分析的本地 Git 项目。
- 默认使用本地执行环境；项目路径和 Git ref 必须长期存在。
- 工作日和时间由用户配置，例如周一至周五 `09:30`。
- 默认回看最近七天，长时间关机后可以临时扩大窗口手动补跑。

首次配置提示：

```text
使用 $dagojiao 帮我完成首次配置。输入来源是飞书群，输出到飞书知识库；请先检查授权，再创建工作日定时任务，最后带我做一次重复链接验收。
```

Skill 会读取 `templates/automation-prompt.md`，替换群名称、Wiki URL、项目路径、Git ref、时区和回看天数，并使用 Codex 的自动化能力创建或更新任务。

## 本机运行边界

本地计划任务需要：

- 电脑在执行时间开机并保持唤醒。
- Codex Desktop 正在运行。
- 本地项目仍存在且可读。
- 飞书授权、网络和代理可用。

如果电脑错过执行时间，不要假设任务一定会补跑。下一次计划运行会在回看窗口内重新发现未处理消息；超过窗口时手动扩大回看天数。

## 通知与结果

- 成功：报告成功数、跳过数、待重试数和目标分类。
- 授权、网络或权限失败：报告失败阶段、原因、用户动作和重试语义。
- 证据不足：保留为待重试，不生成笔记。
- macOS 通知和飞书群回执都是附加提醒；Codex「已安排」结果是可靠记录。

## 权限模式

计划任务无人值守运行，应使用满足飞书网络访问和项目只读分析的最小权限。不要为了省事授予不相关目录写权限或管理员权限。公开 Skill 不修改本地项目文件。
